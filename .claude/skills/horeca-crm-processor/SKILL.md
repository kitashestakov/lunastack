# /horeca-crm-processor — Запись апдейтов по CRM-лиду

## Audience & Preamble Overrides

Скилл рассчитан на **sales rep'ов HoReCa-вертикали Luna Pastel**, не на рекрутеров. Из-за этого две секции preamble применяются иначе:

- **`Config` секция — НЕ применяется.** Этот скилл НЕ читает `~/.luna-stack/config.yaml` и НЕ требует `/onboarding`. Sales rep может никогда не настраивать Luna Stack — нужен только подключенный Notion-коннектор в Claude Desktop и доступ к HoReCa CRM в Notion. Если файл конфига существует — игнорируй его.
- **`Session Binding` секция — НЕ применяется.** Этот скилл не работает с Vacancies-базой и не требует активной вакансии. Каждый запуск — отдельный лид (или несколько подряд через Step 7).

Все остальные правила preamble (язык, типография, безопасность, AskUserQuestion-формат) — применяются как обычно.

## Execution Mode

**Скилл выполняется немедленно, без предварительного плана.** Sales rep уже одобрил выполнение запуском `/horeca-crm-processor`. Не показывай launch plan, не используй EnterPlanMode, не дублируй approve.

**Подтверждение есть только в одном месте — превью комментария перед записью в Notion (Step 5).** Это override на preamble правило «Always confirm before writing to Notion»: подтверждение происходит ровно раз, на превью, не отдельно за каждое поле.

## Output Language

Весь output скилла в чате — **на русском**, по правилам preamble (типография, «елочки», без буквы ё кроме случаев смыслоразличения).

**Транскрипты голосовых и видео — на языке оригинала.** Если sales rep записал голосовое на испанском — транскрипт остается на испанском. Финальный комментарий и его саммари в чате — на русском, но дословные цитаты внутри комментария сохраняй на исходном языке.

## Step 0: Tooling Check & Auto-Install (только при первом запуске в сессии)

Скилл использует **ffmpeg** (нормализация аудио, извлечение звука из видео-кружков) и **whisper.cpp** (локальная транскрипция, бинарь после `brew install whisper-cpp` называется `whisper-cli`). Модель — `ggml-base.bin` в `~/.luna-stack/whisper/`.

**Sales rep ничего не ставит руками.** Скилл сам вызывает `brew` и `curl`. Все нужные разрешения (brew, узкий curl на `~/.luna-stack/whisper/`, домены huggingface и brew CDN) уже прописаны в `.claude/settings.json` этого проекта.

Проверь наличие тремя отдельными командами (составная команда хуже matches под permission patterns):

```bash
command -v ffmpeg
command -v whisper-cli
ls ~/.luna-stack/whisper/ggml-base.bin
```

**Если все три вернули корректный путь** (без «not found» / «No such file») — продолжай молча. Запомни на сессию, чтобы не проверять при каждом следующем лиде.

**Если чего-то не хватает** — сообщи sales rep'у одной строкой:

«Один раз поставлю ffmpeg + whisper.cpp + модель транскрипции. Минута-две, дальше всё мгновенно.»

Затем выполни последовательно (только то, чего реально не хватает):

### 0a. Brew tooling

Если `command -v ffmpeg` или `command -v whisper-cli` не нашли бинарь:

```bash
brew install ffmpeg whisper-cpp
```

`brew install` идемпотентен — если ffmpeg уже стоит, а whisper-cpp нет, он поставит только whisper-cpp без переустановки ffmpeg. Можно безопасно вызывать всегда, когда хотя бы одного нет.

### 0b. Whisper model

Если `ls ~/.luna-stack/whisper/ggml-base.bin` вернул «No such file»:

```bash
mkdir -p ~/.luna-stack/whisper
curl -L -o ~/.luna-stack/whisper/ggml-base.bin "https://huggingface.co/ggerganov/whisper.cpp/resolve/main/ggml-base.bin"
```

Модель ~150MB, скачивается за десятки секунд на нормальном интернете.

### 0c. Re-check

Повтори те же три проверки из начала Step 0. Все три должны вернуть путь без «not found».

### 0d. Failure modes

Если что-то всё-таки упало — конкретная диагностика, не общий «попроси установить вручную»:

- **`brew: command not found`** → Homebrew не установлен. Сообщи sales rep'у:

  «У тебя нет Homebrew (это менеджер пакетов macOS, нужен для установки ffmpeg и whisper). Открой страницу https://brew.sh, скопируй команду установки в Терминал, дождись окончания (5-10 минут) и снова запусти `/horeca-crm-processor`.»

  Не пытайся ставить brew сам — это требует sudo и интерактивных подтверждений в Терминале.

- **`brew install` упал с ошибкой** → покажи stderr-сообщение целиком. Если в нём упоминается «permission denied» или «failed to update tap» — попроси sales rep'а запустить `brew update && brew doctor` в Терминале.

- **`curl` для модели упал** → проверь сетевой доступ (`curl -I https://huggingface.co` для диагностики), попробуй ещё раз. Если падает повторно — сообщи: «Не получается скачать модель. Проверь интернет и снова запусти `/horeca-crm-processor`.»

- **После повторной проверки чего-то всё ещё нет** — сообщи sales rep'у конкретно чего не хватает и предложи перезапустить скилл. Не продолжай — без транскрипции скилл не работает (degraded mode не предусмотрен).

Когда все три проверки прошли — продолжай молча, без анонса «готово». Sales rep увидит, что ты сразу перешла к Step 1.

## Step 1: Collect Inputs

Скилл может получить вход двумя способами:

**(a) Чистый вызов** — `/horeca-crm-processor` без аргументов. Перейди к запросу.

**(b) Вызов с данными** — sales rep уже включил в первое сообщение ссылку на лида и/или обновления (текст, изображения, файлы). Разбери что есть, переходи к Step 2.

В случае (a) выведи как plain text (НЕ AskUserQuestion):

```
Скинь ссылку на лида в Notion и обновления одним сообщением. Можно мешать любые форматы:

- текст переписки или своими словами
- скриншоты переписки, визитки, меню (drag-drop в чат)
- голосовые .ogg / .mp3 / .m4a / .wav
- видео-кружки .mp4 / .mov / .webm

Голосовые и кружки я транскрибирую сама.
```

Wait for the user message. Затем разбери:

- **URL лида:** ищи `notion.so/...`, `notion.site/...` или голый 32-символьный hex / UUID. Если URL несколько — возьми первый и упомяни в Step 5 что использовала первую ссылку. Если URL не нашелся — попроси повторно: «Не вижу ссылки на лида. Скинь URL Notion-страницы.»
- **Текст:** все что не URL и не файл — в текстовый блок обновлений.
- **Изображения:** если Claude Desktop вставил их встроенно — анализируй прямо. Если дан только путь к файлу — `Read /path/to/image.<ext>`.
- **Аудио / видео файлы:** собери пути для Step 3c / 3d. `~/Downloads` уже разрешён в `.claude/settings.json` — туда Telegram сохраняет файлы по дефолту, поэтому большинство случаев работают без вопросов. Если файл лежит вне `~/Downloads` — просто пытайся прочитать его как обычно. Claude Code сам обработает permission flow (см. handler ниже).

### Permission handler (для файлов из неразрешённых папок)

Когда tool пытается прочитать файл из папки, на которую sandbox не выдал доступ, **сам Claude Code показывает sales rep'у нативный диалог macOS** «Allow Claude Code to access this folder?». Sales rep нажимает Allow — доступ открывается. Скиллу ничего делать не нужно.

Поведение скилла:

1. **Просто делай `Read /path/to/file`** как обычно. Не пытайся сам редактировать `.claude/settings.json`, не угадывай папки заранее, не блокируй на AskUserQuestion перед чтением.

2. **Если чтение упало** с sandbox-ошибкой («Operation not permitted» / «Permission denied») и Claude Code не показал sales rep'у диалог (или sales rep его пропустил / нажал Deny):

   - Сообщи в чате одной строкой: «Не получилось прочитать `[имя файла]` — нужен доступ к папке `[папка]`. Самый простой способ: перетащи файл в "Загрузки" через Finder и снова перетащи в окно Claude Code.»
   - Жди новый attachment ИЛИ переход к Step 7 (если sales rep решает пропустить).

3. **Если sales rep так и не дал файл** после второй попытки — отметь в Step 4 в комментарии: «Файл `[имя]` пропущен (не удалось получить доступ).» И продолжай с остальными источниками. Не блокируй обработку всего лида из-за одного недоступного файла.

**Что НЕ делать:**
- Не редактируй `.claude/settings.json` сам — это не работает интерактивно (изменения подхватываются непредсказуемо) и пугает sales rep'а ненужной технической подробностью.
- Не проси sales rep'а вручную прописывать пути или копировать файлы через Терминал. Sales rep работает только в окне Claude Code и Finder.

## Step 2: Extract Lead Context

Извлеки `page_id` из URL. Notion URL может быть в форматах:

- `https://www.notion.so/Page-Title-32abc...` — ID это последние 32 hex после последнего дефиса.
- `https://www.notion.so/workspace/32abc-...` — то же.
- `https://www.notion.so/32ef9167-2e00-...-000cb6fb546c` — UUID с дефисами.

Проще всего — передать MCP-инструменту полный URL, он сам разберет.

Загрузи свойства страницы:

```
mcp__claude_ai_Notion__notion-fetch
  id: "<URL or page_id>"
```

**Валидация: это страница HoReCa CRM лида?** Ожидай, что в свойствах есть **Stage** (значения Cold / Replied / Meeting Planned / Meeting held / Proposal / Won) и **Company Name** (или «Название компании»). Дополнительно: Contact Name, Notes, Source, Category, Next Action.

Если ни Stage, ни Company Name не найдены — останавливайся:

«Это не похоже на страницу лида в HoReCa CRM (нет полей Stage / Company Name). Проверь ссылку — может быть это страница вакансии или другой базы.»

**Опциональный контекст для лучшего саммари:** загрузи существующие комментарии страницы:

```
mcp__claude_ai_Notion__notion-get-comments
  page_id: "<page_id>"
```

Прочитай последние 1-3 комментария. Это поможет:
- не дублировать факты, которые уже зафиксированы (например, имя контакта, бюджет)
- понять текущий контекст истории общения с лидом
- увидеть последний follow-up и ссылаться на него если уместно

Не цитируй существующие комментарии в новом — только используй для контекста.

## Step 3: Process Updates

Создай рабочую temp-папку для текущего лида:

```bash
mkdir -p ~/.luna-stack/tmp/processor
```

Это разрешенная sandbox-зона записи. Все промежуточные файлы (.wav, .txt) кладутся сюда. Не пытайся писать в `$TMPDIR` или `/tmp` — sandbox их не разрешает в этом проекте.

Для каждого типа входящего материала:

### 3a. Текст

Используй как есть. Если это форвард из Telegram (видны метки времени, имена отправителей) — сохрани структуру «кто кому что» при формировании ключевых фактов.

### 3b. Изображения

Если Claude видит изображение нативно (встроенный image content в сообщении) — анализируй прямо.

Если дан только путь к файлу:

```
Read /path/to/screenshot.png
```

Для каждого изображения определи тип и извлеки информацию:

- **Скриншот переписки** (Telegram, WhatsApp, Instagram DM, email) → процитируй ключевые сообщения, кто их отправил, время если видно. Это самый ценный источник фактов.
- **Визитка** → имя, должность, компания, контакты (телефон, email, IG handle).
- **Меню / интерьер / витрина / пост в соцсети** → краткое описание концепции, что отличает заведение, что важно для понимания клиента.
- **Документ / счет / бриф** → ключевые цифры, даты, условия.

### 3c. Аудио (голосовые .ogg, .mp3, .m4a, .wav)

Для каждого файла (N — порядковый номер 1, 2, 3...):

```bash
# Конвертация в whisper-friendly формат: 16kHz mono PCM WAV
ffmpeg -i "<input_path>" -ar 16000 -ac 1 -c:a pcm_s16le ~/.luna-stack/tmp/processor/audio_N.wav -y -loglevel error

# Транскрипция с автоопределением языка
whisper-cli -m ~/.luna-stack/whisper/ggml-base.bin -f ~/.luna-stack/tmp/processor/audio_N.wav -otxt -of ~/.luna-stack/tmp/processor/audio_N -l auto

# Прочитай результат
cat ~/.luna-stack/tmp/processor/audio_N.txt
```

Если ffmpeg или whisper-cli вернули ненулевой exit code — отметь в Step 4 в комментарии: «Аудио N не удалось транскрибировать ([краткая причина из stderr]).» Не блокируй обработку остальных файлов и не возвращайся в Step 0 — установка уже сделана при первом запуске сессии. Если whisper падает массово (несколько файлов подряд) — попроси sales rep'а проверить файлы (возможно битые) и продолжай с того что есть.

### 3d. Видео (.mp4, .mov, .webm — кружки из Telegram)

Извлеки только аудиодорожку — визуальный ряд кружка обычно не несет фактов (лицо говорящего):

```bash
ffmpeg -i "<input_path>" -vn -ar 16000 -ac 1 -c:a pcm_s16le ~/.luna-stack/tmp/processor/video_audio_N.wav -y -loglevel error

whisper-cli -m ~/.luna-stack/whisper/ggml-base.bin -f ~/.luna-stack/tmp/processor/video_audio_N.wav -otxt -of ~/.luna-stack/tmp/processor/video_audio_N -l auto

cat ~/.luna-stack/tmp/processor/video_audio_N.txt
```

Если sales rep явно скажет, что в видео есть важный визуал (показал документ, локацию, продукт) — попроси описать словами или приложить скриншот кадра отдельно. Сам кадры не извлекай.

### 3e. Temp files lifecycle

Cleanup НЕ нужен. Файлы в `~/.luna-stack/tmp/processor/` перезаписываются при следующем запуске благодаря `-y` флагу ffmpeg и стабильным именам (`audio_1.wav`, `audio_2.wav`, `video_audio_1.wav` и т.д.). Размер минимальный (мегабайты на лида). Sandbox запрещает `rm` — не пытайся удалять, оно упадет.

Используй стабильные имена в пределах одного лида: счетчик N сбрасывается при каждом новом лиде в Step 7. То есть для нового лида первый аудиофайл снова `audio_1.wav` и перезаписывает старый.

## Step 4: Compose Comment

Слей все обработанные источники в **один связный markdown-комментарий** по структуре:

```
**Обновление DD.MM.YYYY**

[1-3 предложения краткого саммари: что произошло на этом касании, общий вектор. Без оценок, без продажного тона. Это служебный комментарий для команды.]

**Ключевые факты:**
- [конкретный факт: цифра, имя, дата, возражение, договоренность]
- [конкретный факт]
- [...]

**Договоренности и follow-up:**
- [explicit: кто кому что когда. Например: «Жду от Marc список ролей до пятницы 02.05» / «Договорились созвониться на следующей неделе» / «Лид взял паузу до конца месяца»]

[Опционально, если есть длинный транскрипт переписки или голосового, важный для нюансов:]

**Дословно:**
> [цитата на исходном языке]
> [цитата]
```

**Правила содержания:**

- **Дата** — сегодняшняя в формате `DD.MM.YYYY` (бери из системы).
- **Саммари** — фактическое, нейтральное. Не «лид в восторге» — а «лид сказал, что это решает X». Не «отличная встреча» — а «обсудили N тем за 30 минут».
- **Ключевые факты** — конкретика. Цифры, имена, даты, конкретные возражения. Плохо: «обсудили условия». Хорошо: «бюджет до €4K/мес, не готов на €5K, спрашивал про скидку за 6 мес commit».
- **Договоренности** — explicit. Кто что когда. «Жду от него список ролей до пятницы 02.05» лучше чем «обещал прислать роли». Если sales rep ничего не должен и лид ничего не должен — пиши «открытых обязательств нет, ждем инициативы лида».
- **Дословно** — добавляй ТОЛЬКО если в переписке/голосовом есть формулировка, которую важно сохранить точно (возражение, тон, эмоциональная реакция, конкретный запрос). Не дублируй то что уже в саммари. На исходном языке.
- **Без AI-boilerplate**: «Подводя итог», «Стоит отметить», «В рамках обсуждения», «Хочется отметить» — выжигаем.
- **Без emoji** в тексте комментария (только markdown bold для структуры). Это служебная заметка, не сообщение клиенту.
- **На русском**, кроме блока «Дословно» (исходный язык) и собственных имен / названий компаний.
- **Типография по preamble**: без буквы ё кроме случаев смыслоразличения, «елочки» для русских кавычек, "double quotes" для английских вкраплений.
- **Без подписи автора** — Notion-коннектор подпишет комментарий аккаунтом sales rep'а автоматически.

**Если апдейт пустой / без новых фактов** (sales rep написал что-то вроде «созвон не состоялся, перенесли») — короткое саммари с этим единственным фактом и одной строкой в follow-up. Не выдумывай содержание которого не было.

**Если в обновлениях есть противоречия** между источниками (например, в переписке одно число, в голосовом другое) — отметь оба и пометь «уточнить у лида».

## Step 5: Preview & Confirm

Покажи sales rep'у сформированный комментарий **полностью** в чате как markdown-блок:

```
Вот что записываю на странице лида [Company Name]:

---

[полный текст комментария из Step 4]

---
```

Затем AskUserQuestion:

- question: «Записать в Notion?»
- header: «Подтверждение»
- options:
  - label: "Записать (Рекомендуется)"
    description: "Комментарий уйдет на страницу лида как есть"
  - label: "Поправить"
    description: "Скажи что изменить — перегенерирую и покажу заново"
  - label: "Отменить"
    description: "Ничего не записываю, выходим"

Если «Поправить» — спроси что менять (plain text), пересобери комментарий, снова покажи и спроси. Iterate до «Записать» или «Отменить».

Если «Отменить» — пропусти Step 6, перейди сразу в Step 7 («Обработать следующий лид?»).

## Step 6: Write Comment

```
mcp__claude_ai_Notion__notion-create-comment
  page_id: "<lead page id from Step 2>"
  rich_text: [{ text: { content: "<final comment from Step 4 with edits from Step 5>" } }]
```

## Notes

- Этот скилл **не требует** `~/.luna-stack/config.yaml` и `/onboarding`. Sales rep может не настраивать Luna Stack вообще.
- Этот скилл **не работает** с Huntflow. HoReCa CRM — это sales-pipeline, не recruitment.
- Этот скилл **пишет только комментарий** на странице лида. Никаких изменений свойств (Stage, Notes, Next Action, Claude-колонок) — это территория Насти и `/horeca-crm-plan`. Если sales rep'у нужно поменять Stage — он делает это руками в Notion.
- **Авторство комментария** определяется Notion-коннектором — комментарий идет от лица аккаунта, к которому подключен Notion в Claude Desktop. Подпись внутри текста не нужна.
- **Качество транскрипции** на `ggml-base.bin` приемлемое для русского, испанского, английского. Если sales rep работает в основном на каталанском или редких языках — рекомендуй `ggml-medium.bin` (~1.5GB) или `ggml-large-v3.bin` (~3GB) при первом feedback'е.
- **Совместимость с `/horeca-crm-plan`:** plan читает все комментарии страницы при обработке (Step 2d Item 1 в horeca-crm-plan). Так что комментарий, записанный этим скиллом, автоматически учитывается при следующем прогоне plan для обновления Claude-колонок.
