# /meet — Назначение звонка

Дизайн-цель: 1-2 шага диалога. Всё что можно — берётся из контекста (вакансия в session-binding, воронка в Хантфлоу, карточка клиента в Notion, конфиг рекрутера).

## Pre-check 0: Google Calendar connector

Этот скилл требует подключённого Google Calendar в Claude Desktop. Тулы коннектора видны в этой сессии — посмотри список доступных tool calls и найди группу для Google Calendar. Имена тулов — `create_event`, `list_events`, `list_calendars`, `update_event`, `respond_to_event`, `suggest_time`, `get_event`, `delete_event`. У каждого рекрутера префикс может отличаться (Claude Code присваивает UUID per-installation), поэтому ищи по суффиксу (`*__create_event` и т.д.), а не по полному имени. Используй именно эти тулы — не пытайся ходить через Bash/curl.

**Если коннектор НЕ подключён** (тулов с этими суффиксами в сессии нет) — выведи как plain text и стоп:

```
Не вижу подключённый Google Calendar. Подключи разово:
→ Settings → Connectors → Google Calendar → Connect
→ Выбери свой рабочий аккаунт Luna Pastel
→ Подтверди разрешения (создание событий + чтение для проверки занятости)

Когда подключишь — снова /meet.
```

Не предлагай альтернатив (curl, Bash). Без коннектора скилл не работает.

**Permission popups.** При первом запуске `/meet` Claude Code может показать popup «Allow this tool?» на каждый тул Google Calendar. Это нормально — нажми «Always allow». Claude Code запомнит выбор в `~/.claude/settings.local.json` и больше спрашивать не будет.

## Pre-check: Vacancy session binding

Прочитай `~/.luna-stack/config.yaml` (нужны `email`, `name`, `huntflow_user_id`).

Сессия должна быть привязана к вакансии (правило preamble). Если нет — «Сначала набери /vacancy, чтобы выбрать вакансию.» и стоп.

Из контекста session-binding нужны:
- Notion page ID вакансии
- Huntflow vacancy ID
- Название позиции (для title ивента)
- Имя клиента (для контекста и для интервью с клиентом)
- Текущая стадия вакансии (для дефолта типа звонка)

## Step 1: Parse Intent (намерение в одну строку)

Если в сообщении рекрутера после `/meet` есть текст — это и есть намерение, парсь его. Например:
- `/meet скрининг с Иваном завтра 15:00`
- `/meet интервью с клиентом сегодня в 17:30 на час`
- `/meet брифинг с клиентом завтра 10:00 по Барселоне`

**Если после `/meet` ничего нет** — выведи plain text и stop:

```
Кратко: тип звонка + кандидат + когда. Например:
• «скрининг с Иваном завтра 15:00»
• «интервью с клиентом сегодня в 17:30 на час»
• «брифинг с клиентом завтра 10:00»
```

Жди ответ рекрутера в виде свободного текста. **НЕ** используй AskUserQuestion на этом шаге.

## Step 2: Извлеки поля

Из строки намерения вытащи:

| Поле | Как извлекаешь / дефолт |
|---|---|
| **Тип звонка** | Из ключевых слов: «скрининг», «интервью с клиентом» / «interview», «брифинг», «техническое» / «tech», «финал». Если не указан — дефолт по стадии вакансии (см. таблицу ниже). |
| **Длительность** | Если указано в строке («на час», «30 мин», «полтора часа») — используй. Иначе — дефолт по типу (таблица ниже). |
| **Дата + время** | Парсь natural language. Поддерживай: «завтра», «сегодня», «послезавтра», «в понедельник», «через час», конкретные даты. Время — 24-часовой формат и 12-часовой («3 дня» = 15:00). |
| **Таймзона** | **Дефолт — Europe/Madrid (Барселона).** Если рекрутер указал TZ в строке («по Тбилиси», «MSK», «Moscow time», «UTC+3», «по Москве») — используй её. Распознавай и города, и оффсеты. |
| **Кандидат / клиент** | По имени или email. Для брифинга — `client` (без кандидата). |
| **Доп. участники** | Если в строке есть emails или явное «и пригласи X» — добавь. Иначе пропусти, добавится в Step 6 «Изменить». |

**Дефолты по типу звонка:**

| Тип | Длительность | Кого приглашаем (помимо рекрутера и `ba@lunapastel.io`) |
|---|---|---|
| Скрининг | 30 мин | кандидат |
| Интервью с клиентом | 45 мин | кандидат + клиент |
| Техническое интервью | 60 мин | кандидат + клиент |
| Финал | 60 мин | кандидат + клиент |
| Брифинг с клиентом | 60 мин | клиент |
| Другое (если рекрутер так назвал) | 30 мин | кандидат |

**Дефолт типа по стадии вакансии:**

| Стадия | Дефолт типа звонка |
|---|---|
| Research | Брифинг с клиентом |
| Active (воронка пустая) | Скрининг |
| Active (есть кандидаты на «Client Review») | Интервью с клиентом |
| Active (есть кандидаты на «Initial Interview») | Техническое интервью / Финал |
| Прочее | Скрининг |

## Step 3: Резолв кандидата (пропусти для брифинга)

Если тип — Скрининг / Интервью с клиентом / Техническое / Финал, нужен email кандидата.

```bash
scripts/huntflow.sh applicants-list <huntflow_id>
```

Найди кандидата fuzzy-матчем по имени из Step 2 (поддерживай транслитерацию: «Иван» ↔ «Ivan», «Наташа» ↔ «Natalia»).

- **1 совпадение** → бери email, имя, applicant_id из Хантфлоу. Молча, без вопросов.
- **2+ совпадений** → AskUserQuestion с топ-3 вариантами + «Это другой человек».
  - Формат опций: `[Имя] — [Stage] · [email]`
- **0 совпадений** → plain text: «Не нашла "[имя]" в воронке. Email кандидата?» Жди ответ — это будет email или полное имя.
  - Если email — используй и помечай applicant_id как None (Хантфлоу не обновится в Step 8, но ивент создастся).

## Step 4: Резолв клиента (для всех типов кроме Скрининг)

Для типов Интервью с клиентом / Техническое / Финал / Брифинг с клиентом нужен email клиента.

1. Из Notion-карточки вакансии возьми поле «Клиент» — это relation на карточку клиента. Подгрузи карточку клиента:

```
mcp__claude_ai_Notion__notion-fetch
  id: "<client page URL or ID>"
```

2. Поищи в properties клиента поля типа `Email`, `email`, или текст-поле с emails в content страницы.

3. **Если email найден (1 шт)** → используй молча.
4. **Если несколько emails** → AskUserQuestion: «У клиента в Notion несколько контактов. Кого приглашаем?» + опции.
5. **Если email НЕ найден** → plain text: «В карточке клиента "[Client Name]" нет email. Кому из клиента отправить инвайт? (один или несколько email через запятую)» Жди ответ.

## Step 5: Проверка занятости

Через Google Calendar коннектор — get список ивентов рекрутера в окне `[start, end]` нового события.

- **Если конфликта нет** → молча идём дальше.
- **Если есть пересечение** → AskUserQuestion:
  - question: «У тебя в это время уже стоит "[Conflict event title]". Что делаем?»
  - header: «Конфликт»
  - options:
    - «Поставить параллельно (Рекомендуется если параллельно ок)»
    - «Сдвинуть на [start + duration] (Рекомендуется если нет)» — например, если 15:00–15:30 занято и новый звонок 30 мин, предложи 15:30–16:00
    - «Сдвинуть на час позже»
    - «Отмена»

Подсказка по дефолту: если конфликт — обычно «Сдвинуть» лучше, чем «Параллельно». Но финальное решение за рекрутером.

## Step 6: Превью + единственное подтверждение

Покажи структурированный блок:

```
📅 [Тип] · [Имя кандидата или клиент] · [Позиция] · Luna Pastel
🕐 [День недели], [DD месяц], HH:MM–HH:MM ([таймзона])
👥 Участники:
   • [recruiter email] (ты)
   • [candidate / client email] ([роль: кандидат / клиент])
   • [client email] (клиент) — если интервью с клиентом
   • ba@lunapastel.io (tldv)
🎥 Google Meet ссылка создастся автоматически
📋 В описание положу: ссылку на карточку вакансии в Notion + ссылку на профиль кандидата в Хантфлоу
```

**Затем AskUserQuestion** (единственный обязательный вопрос всего флоу):

- question: «Создать ивент?»
- header: «Подтверждение»
- options:
  - «Создать (Рекомендуется)»
  - «Изменить» — plain text «Что поправить? Напиши одной строкой (например: "перенеси на 16:00", "добавь ceo@client.com", "убери ноутейкера", "длительность 45 мин").» — применяй изменения, перепоказывай превью.
  - «Отмена»

## Step 7: Создание ивента

Используй create-event тул Google Calendar коннектора. Поля:

- **summary**: `[Тип] · [Имя кандидата или Client Name] · [Позиция] · Luna Pastel`
  - Примеры:
    - `Скрининг · Иван Петров · Head of Online Sales · Luna Pastel`
    - `Интервью с клиентом · Иван Петров · Head of Online Sales · Luna Pastel`
    - `Брифинг с клиентом · Neginski · Head of Online Sales · Luna Pastel`
- **start**: ISO datetime + timezone (например `2026-05-08T15:00:00`, `Europe/Madrid`)
- **end**: ISO datetime + timezone
- **attendees**: список email-ов из Step 3-4 + `ba@lunapastel.io` + email рекрутера из конфига
- **conferenceData**: создать новый Google Meet (тул коннектора обычно принимает флаг `createConference: true` или похожий — используй то, что есть)
- **description**: 
  ```
  Тип: [Тип]
  Вакансия: [Position] для [Client Name]
  Карточка вакансии: [Notion URL]
  Профиль кандидата: [Huntflow applicant URL] (если применимо)

  Заметки:
  • Запись и транскрибт делает tldv (через ba@lunapastel.io)
  • После звонка — /calls для сохранения транскрибта
  ```
- **reminders**: дефолтные коннектора (обычно email + popup за 30 минут)
- **sendUpdates**: «all» — чтобы инвайты улетели всем участникам

Если коннектор требует выбрать calendar (primary vs secondary) — используй `primary`.

## Step 8: Обновление Хантфлоу (если применимо)

Маппинг типа → Huntflow status:

| Тип | Status ID | Status Name |
|---|---|---|
| Скрининг | 3940 | Screening |
| Интервью с клиентом | 3941 | Initial Interview |
| Техническое интервью | 3942 | Technical Interview |
| Финал | (без изменений) | — |
| Брифинг с клиентом | (нет кандидата, скипаем) | — |

Если applicant_id есть и тип маппится:

```bash
scripts/huntflow.sh applicant-move <applicant_id> <huntflow_vacancy_id> <status_id>
```

Если не получилось — не блокируй флоу, сообщи: «Ивент создан, но не получилось обновить статус в Хантфлоу — обнови вручную.»

**Комментарий в Notion** — в карточку вакансии добавь короткую заметку через notion-create-comment:

```
mcp__claude_ai_Notion__notion-create-comment
  page_id: "<vacancy page ID>"
  rich_text: [{ text: { content: "📅 [Тип] · [Имя]: [Дата DD.MM HH:MM TZ]\nMeet: <URL>" } }]
```

## Step 9: Готово

Покажи рекрутеру:

```
Готово ✓

📅 Ивент создан, инвайты ушли [Имя кандидата]/[Имя клиента] и ba@lunapastel.io
🕐 [DD.MM HH:MM TZ]
🎥 Meet: [URL]

→ [Открыть в календаре]([event URL])
```

## Step 10: Что дальше

Use AskUserQuestion:

- question: «Что дальше?»
- header: «Дальше»
- options (релевантные по типу):
  - Для Скрининг: «Подготовить кандидата (/screening для прокола профиля)» / «Закончить»
  - Для Интервью с клиентом: «Подготовить кандидата к интервью (читаю гайд)» / «Закончить»
  - Для Брифинг: «Подготовить вопросы (/briefing)» / «Закончить»
  - Всегда: «Поставить ещё один звонок»

## Edge cases — обрабатывай молча

- **Время «через час»** — резолв в реальный datetime от now() в Europe/Madrid (или TZ из строки).
- **«полдень», «вечером»** — полдень = 12:00, вечер = уточни plain text «Какое конкретное время вечером?»
- **Дата без года** — текущий год. Если получилась дата в прошлом — следующий год.
- **Прошедшее время («сегодня в 9:00» когда уже 14:00)** — переспроси: «Время уже прошло. Имела в виду завтра 9:00?»
- **TZ-ambiguity без явного указания** — Europe/Madrid (Барселона), это дефолт.
- **Кандидат без email в Хантфлоу** — спроси email plain text.
- **Ивент длиннее 4 часов** — переспроси: «4+ часа на встречу — точно? Может, имела в виду 4 минуты?»

## Что скилл НЕ делает (явно по дизайну)

- ❌ Не спрашивает «какой длительности?» — берёт дефолт по типу.
- ❌ Не спрашивает «добавить ноутейкера?» — всегда `ba@lunapastel.io`.
- ❌ Не спрашивает «создать Meet?» — всегда да.
- ❌ Не спрашивает «отправить инвайты?» — всегда да (`sendUpdates: all`).
- ❌ Не спрашивает «куда сохранить?» — всегда GCal + Хантфлоу + комментарий в Notion.
- ❌ Не парсит сложные RRULE/recurring — только разовые ивенты. Для регулярных встреч — отправь рекрутера в Google Calendar UI.
- ❌ Не отменяет/не переносит существующие ивенты в этой версии. Для отмены — Google Calendar UI (отдельный скилл `/reschedule` можем сделать позже).
